# Code Patterns & Conventions

## Media Download Pipeline

```
User sends URL
  → url-detector.ts: detectMediaUrl() → { platform, url }
  → text-input-handler.ts: platform-specific routing
      ├── YouTube/TikTok/Facebook (admin only) → fetch info → show picker → setAdminState()
      └── Others (admin + guest) → downloadAndSendMedia() directly
  → download-and-send.ts: downloadAndSendMedia()
      ├── downloadMedia() in media-downloader.ts (btch API, 4 servers)
      └── sendMedia() in send-media.ts
```

## btch API Server Failover

```typescript
const BTCH_SERVERS = ['https://backend1.tioo.eu.org', ...]; // 4 servers

async function btchFetch(endpoint: string, url: string) {
    for (const server of BTCH_SERVERS) {
        try {
            const res = await fetch(`${server}/api/downloader/${endpoint}?url=...`);
            if (res.status >= 500) continue;  // try next server on 5xx
            if (!res.ok) throw new Error(...); // fail fast on 4xx
            return await res.json();
        } catch (err) {
            if (err.name === 'TimeoutError' || err.message?.includes('returned 5')) continue;
            throw err; // propagate non-retryable errors
        }
    }
}
```

## URL-First Telegram Send Strategy

`send-media.ts` tries passing CDN URLs to Telegram's Bot API directly (no bytes through Worker).

If Telegram rejects with `TelegramUrlFetchError`:
1. Show interactive buttons: `[📥 Download] [❌ Cancel] [📤 Send to @urluploadxbot]`
2. Store `directMediaUrl` in KV under `telegram:state:{userId}`
3. `dl:confirm` callback downloads bytes and re-uploads

Files >50MB: Show URL + @urluploadxbot button (cannot upload via bot API).

## KV State Key Patterns

- Admin state: `telegram:state:{userId}` (TTL 1h, JSON: `AdminState`)

```typescript
interface AdminState {
    action: 'downloading_media';
    context?: {
        downloadUrl?: string;
        downloadPlatform?: string;
        qualities?: Array<{ quality: string; url: string; size?: string }>;

}
```

## Platform-Specific Download Strategy

| Platform | Primary endpoint | Fallback | Picker |
|----------|-----------------|----------|--------|
| TikTok | `tiktok` (rich) | `ttdl` | Video/Audio (admin) |
| Instagram | `aio` | `igdl` | None (auto) |
| Twitter/X | `aio` | `twitter` | None (auto) |
| YouTube | `aio` | `youtube` | Quality list (admin) |
| Facebook | `aio` | `fbdown` | HD/SD (admin, if multi-quality) |
| Threads | `aio` | `threads` | None (auto) |
| SoundCloud | `soundcloud` | — | None (auto, audio) |
| Spotify | `spotify` | — | None (auto, audio) |
| Pinterest | `aio` | `pinterest` | None (auto) |

## Callback Data Format

- `dl:video` — download as video
- `dl:audio` — download as audio
- `dl:hd` — download HD quality (Facebook)
- `dl:sd` — download SD quality
- `dl:yt:{quality}` — download specific YouTube quality (e.g., `dl:yt:1080p`)
- `dl:confirm` — confirm download from stored `directMediaUrl` in KV

## Error Handling Patterns

### Double-Fault Protection
```typescript
} catch (err) {
    console.error('Primary operation failed:', err);
    try {
        await notifyUser(err);
    } catch (notifyErr) {
        console.error('Notification also failed:', notifyErr);
    }
}
```

Applied in: `bot-factory.ts` global error handler (callback and message paths).

### Defensive JSON.parse for KV
```typescript
const raw = await getCached(kv, key);
if (!raw) return null;
try {
    return JSON.parse(raw);
} catch (err) {
    console.error(`[KV] Corrupted state for ${key}:`, err);
    return null;
}
```

### Media Type Detection

`detectMediaType(url)` in `media-downloader.ts`:
- `rapidcdn.app` URLs → decode JWT payload to check filename extension
- URLs with `.jpg/.jpeg/.png/.webp/.heic/.gif` → `photo`
- Everything else → `video`

## TypeScript Conventions

- Strict mode enabled
- `Env` type from `worker-configuration.d.ts` (generated by `npm run cf-typegen`)
- `TELEGRAM_WEBHOOK_SECRET` must be manually added to `worker-configuration.d.ts` (wrangler types doesn't include secrets not in .dev.vars)
- KV binding accessed as `env.DOWNLOAD_CACHE` — binding name is `DOWNLOAD_CACHE` (not `CACHE`)
